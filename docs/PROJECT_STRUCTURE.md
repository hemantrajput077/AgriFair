# Project Structure

## Clean & Simple Structure

```
agrifair/
├── backend/              # Spring Boot backend
│   ├── src/              # Source code
│   ├── uploads/          # User uploads
│   └── pom.xml          # Maven config
│
├── frontend/             # React frontend
│   ├── src/              # Source code
│   ├── public/           # Static assets
│   └── package.json     # Dependencies
│
├── docs/                 # All documentation
│   └── *.md             # Documentation files
│
└── README.md            # Main documentation
```

## Directory Names

- ✅ **Simple**: `backend`, `frontend`, `docs`
- ✅ **Meaningful**: Clear purpose of each directory
- ✅ **No nesting**: Flat structure, easy to navigate
- ✅ **Consistent**: All lowercase, no special characters

## What Changed

1. `Backend/` → `backend/` (lowercase, consistent)
2. `NewFrontend/harat-farm-link-main/` → `frontend/` (flattened, simple)
3. Scattered docs → `docs/` (organized)
4. Removed unnecessary nested directories
5. Removed outdated documentation files
